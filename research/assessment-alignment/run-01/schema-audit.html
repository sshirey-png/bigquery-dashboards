CRITICAL: LHA appears as three different strings across LEAP, anet, and MAP. Green appears as two different strings (anet omits the period and "School"). Any cross-system school filtering must use a CASE statement or lookup table.

Recommended SQL pattern for school normalization:

CASE
  WHEN school_col LIKE '%Ashe%'     THEN 'Ashe'
  WHEN school_col LIKE '%Green%'    THEN 'Green'
  WHEN school_col LIKE '%Hughes%'   THEN 'LHA'
  WHEN school_col LIKE '%Wheatley%' THEN 'Wheatley'
END AS school

9. Join Path Reference

Path A: Direct LASID join (LEAP ↔ anet ↔ MAP ↔ PM)
  leap.LASID = anet.sas_id
  leap.LASID = map.Student_StateID
  leap.LASID = pm.State_StudentNumber
  All STRING. No SAFE_CAST needed.

Path B: Grades/GPAs require roster bridge
  grades.Student_Number = roster.Student_Number
  roster.State_StudentNumber = leap.LASID

Path C: Student_Number join (anet ↔ MAP ↔ PM ↔ Grades)
  anet.sis_id = map.StudentID = pm.Student_Number = grades.Student_Number
  All STRING. No SAFE_CAST needed.

Type mismatches requiring SAFE_CAST:
  Grade_Level: INT64 (grades, roster) vs STRING (LEAP, anet, MAP)
  School_Year: INT64 (PM, grades) vs STRING or in-table-name (anet, MAP)
